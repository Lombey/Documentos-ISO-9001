# Holguras de ISO 9001:2015

Flexibilidades que Corvus puede aprovechar sin perder conformidad.

## Lo que NO exige la norma

| Mito | Realidad |
|------|----------|
| Manual de calidad obligatorio | NO desde 2015. MGC es por elección propia |
| Representante de la dirección | NO obligatorio. Responsabilidades pueden distribuirse |
| Procedimientos documentados para todo | Solo donde la norma dice "información documentada" |
| Auditor interno certificado | Competente sí, certificado no |
| Encuestas de satisfacción | Es UN método. Hay otros válidos |
| Acciones preventivas | Eliminadas. Absorbidas por gestión de riesgos |

## Requisitos con Flexibilidad

### 4.4 Sistema de gestión
- No exige formato específico de mapa de procesos
- Interacciones pueden documentarse de cualquier forma

### 6.1 Riesgos y oportunidades
- No exige matriz formal de riesgos
- No exige metodología específica (FODA es válido)
- Puede ser tan simple como una lista priorizada

### 7.1.5 Recursos de seguimiento
- Solo aplica si la medición afecta conformidad del producto
- Software de tickets = recurso de seguimiento? Solo si mide SLAs críticos

### 7.2 Competencia
- Evidencia puede ser: títulos, experiencia, evaluación interna, desempeño
- No exige plan de capacitación formal si las competencias están cubiertas

### 8.4 Proveedores externos
- "Controles según impacto" = proveedores críticos necesitan más control
- Proveedor de café ≠ proveedor de infraestructura cloud

### 9.1.2 Satisfacción del cliente
- Encuestas son UN método
- Válido: análisis de quejas, renovaciones, referencias, feedback informal documentado

### 9.2 Auditoría interna
- Frecuencia "a intervalos planificados" = la organización decide
- Puede ser anual si el sistema es maduro

### 10.2 No conformidad
- Acción correctiva cuando "aplicable"
- NC menores pueden cerrarse sin AC formal si se corrigen

## Aplicación en Corvus

### Alcance limitado = menos requisitos
Al ser solo Soporte/Post-Venta:
- 8.3 Diseño y desarrollo → EXCLUIDO (no desarrollan software)
- 8.5.1 Producción → Interpretado como "prestación del servicio"
- 7.1.5.2 Trazabilidad de medición → Mínimo (no hay producto físico)

### Digital-first = evidencia válida
- Registros en Sheets = información documentada
- Timestamps automáticos = evidencia de fechas
- Historial de cambios = control de versiones
- Notificaciones automáticas = comunicación documentada

### Automatización = cumplimiento consistente
- Apps Script ejecuta procesos = menos dependencia de memoria humana
- Dashboards AppSheet = seguimiento continuo sin reportes manuales
